Contactページは「連絡先を置くだけ」ではなく、**どのような連絡を歓迎しているか・どこから連絡できるか・長期インターンや開発機会への関心**が伝わるように整理しています。

# Contact-details.md

# Contactページ 要件定義

## 1. このドキュメントの目的

このドキュメントは、ポートフォリオサイトの `Contact` ページに掲載する内容・役割・改善方針・実装方針を定義するための要件定義ファイルである。

Contactページは、閲覧者が自分に連絡を取りやすくするためのページである。

単にGitHubリンクだけを置くのではなく、以下を明確にする。

* どのような連絡を歓迎しているのか
* どの連絡手段が使えるのか
* GitHubやメールなど、どこから連絡できるのか
* 長期インターンや開発機会に関心があること
* ポートフォリオを見た人が次にどの行動を取ればよいのか

Contactページは、ポートフォリオサイトを見た閲覧者が最後に到達する可能性があるページである。

そのため、シンプルで分かりやすく、連絡しやすい構成にする。

---

## 2. Contactページの役割

Contactページの役割は、閲覧者に対して「連絡先」と「連絡してよい内容」を分かりやすく提示することである。

特に、以下の閲覧者を想定する。

* 長期インターンの採用担当者
* 学内カンパニーや大学内プロジェクト関係者
* 開発プロジェクトの関係者
* 研究や学習に関心を持った人
* GitHubや制作物を確認したい人

Contactページでは、閲覧者が迷わず以下の行動を取れるようにする。

```text
GitHubを見る
メールで連絡する
SNSから連絡する
Worksページへ戻る
```

---

## 3. Contactページで伝えたいこと

Contactページでは、以下を伝える。

* ポートフォリオを見てくれたことへの簡単なお礼
* Webアプリ開発に関する連絡を歓迎していること
* 長期インターンや開発機会に関心があること
* 学内プロジェクトや研究に関する連絡も歓迎していること
* GitHubで制作物を確認できること
* 必要に応じてメールやSNSから連絡できること

特に、長期インターン応募や開発機会につなげることを考える場合、以下の一文を入れるとよい。

```text
Webアプリ開発・データ分析・AI活用に関する長期インターンや開発機会に関心があります。
```

## 7. Contact Methods

Contact Methodsでは、連絡手段を一覧で表示する。

掲載候補は以下とする。


## 8. GitHub

GitHubは必ず掲載する。

### 8.1 GitHubの役割

GitHubは、単なる連絡先ではなく、制作物やコードを確認してもらうための重要な導線である。

Contactページでは、GitHubを以下のように扱う。

* 制作物のコードを確認できる場所
* 開発履歴を確認できる場所
* ポートフォリオ内のWorksとつながる場所

### 8.2 表示例

```text
GitHub
制作物のコードや開発履歴を掲載しています。

https://github.com/ユーザー名
```

### 8.3 ボタン文言

```text
GitHubを見る
```

または、

```text
View GitHub
```

日本語サイトの場合は、以下を推奨する。

```text
GitHubを見る
```

---

## 9. Email

Emailは、長期インターンや開発機会につながる連絡手段として重要である。

### 9.1 Email掲載の判断

メールアドレスを直接公開してよい場合は、Contactページに掲載する。

メールアドレスを公開したくない場合は、GitHubやSNSへの誘導のみでもよい。

### 9.2 表示例

```text
Email
長期インターン、開発機会、研究・学習に関するご連絡はこちらからお願いいたします。

example@example.com
```

実際のメールアドレスに置き換える。

### 9.3 注意点

メールアドレスを掲載する場合は、スパム対策も考慮する。

例えば、以下のように表記する方法もある。

```text
example [at] example.com
```

ただし、ポートフォリオとして分かりやすさを重視する場合は、通常のメールリンクでもよい。

### 9.4 mailtoリンク

メールアドレスを掲載する場合は、`mailto:` リンクを設定してもよい。

例：

```text
mailto:example@example.com
```

ボタン文言は以下とする。

```text
メールで連絡する
```

## 10. SNS

SNSは、必要に応じて掲載する。

### 10.1 掲載候補

```text
X
LinkedIn
Wantedly
Qiita
Zenn
```

現時点で運用していないSNSは、無理に載せなくてよい。

長期インターン向けであれば、GitHubとEmailを優先する。

### 10.2 表示例

```text
SNS
学習記録や開発に関する発信を行っています。

X: https://x.com/ユーザー名
```

### 10.3 注意点

私的な投稿が多いSNSは、ポートフォリオに載せるか慎重に判断する。

開発・学習・研究に関連した発信をしているSNSのみ掲載することを推奨する。

## 15. 実装方針

Contactページの実装では、以下を意識する。

### 15.1 シンプルにする

Contactページは、情報を詰め込みすぎない。

閲覧者がすぐに連絡手段を見つけられることを最優先にする。

### 15.2 連絡手段をカード形式で表示する

GitHub、Email、SNSなどをカード形式で表示すると見やすい。

例：

```text
GitHub
制作物のコードや開発履歴を掲載しています。
[GitHubを見る]

Email
長期インターンや開発機会に関するご連絡はこちらからお願いいたします。
[メールで連絡する]
```

### 15.3 ボタンを分かりやすくする

ボタン文言は、閲覧者が次に何をするのか分かるようにする。

推奨文言：

```text
GitHubを見る
メールで連絡する
制作物を見る
```

### 15.4 外部リンクの扱い

GitHubやSNSなどの外部リンクは、新しいタブで開くようにしてもよい。

実装時には、以下を考慮する。

```text
target="_blank"
rel="noopener noreferrer"
```

Next.jsでは、外部リンクは通常の `a` タグでもよい。

内部リンクは `Link` コンポーネントを使う。

### 15.5 mailtoリンクの扱い

メールボタンには、必要に応じて `mailto:` を設定する。

例：

```text
mailto:example@example.com
```

メール件名をあらかじめ設定する場合は、以下のような形式も検討できる。

```text
mailto:example@example.com?subject=ポートフォリオサイトを拝見しました
```

ただし、件名を固定しすぎる必要はない。

## 16. データ管理方針

連絡先情報は、今後変更される可能性がある。

そのため、可能であれば定数として管理する。

### 16.1 データ構造例

```ts
const contactLinks = [
  {
    label: "GitHub",
    description: "制作物のコードや開発履歴を掲載しています。",
    href: "https://github.com/your-username",
    buttonText: "GitHubを見る",
    type: "external",
  },
  {
    label: "Email",
    description: "長期インターン、開発機会、研究・学習に関するご連絡はこちらからお願いいたします。",
    href: "mailto:example@example.com",
    buttonText: "メールで連絡する",
    type: "email",
  },
];
```

### 16.2 データ管理のメリット

連絡先を定数化することで、以下のメリットがある。

* GitHub URLを変更しやすい
* Emailを変更しやすい
* SNSを追加しやすい
* Contactカードをコンポーネント化しやすい
* 表示内容の管理が簡単になる

## 20. 注意事項

* Contactページを長くしすぎない
* GitHubリンクだけで終わらせない
* 連絡歓迎の内容を明確にする
* メールアドレスを公開するかどうかは慎重に判断する
* SNSは開発・学習・研究に関係するものだけ掲載する
* 私的なSNSを無理に掲載しない
* 空リンクを表示しない
* 外部リンクの設定を確認する
* スマートフォンでもボタンが押しやすいようにする
* 長期インターン向けに、分かりやすく自然な文章にする

Contactページでは、以下を伝える。

```text
ポートフォリオをご覧いただいた閲覧者が、GitHub・Email・SNSなどを通して連絡しやすい状態にすること。
```

また、単に連絡先を並べるだけでなく、以下を明確にする。

```text
どのような連絡を歓迎しているのか。
どこから連絡できるのか。
制作物はどこで確認できるのか。
長期インターンや開発機会に関心があるのか。
```

Contactページを通して、閲覧者に以下の印象を与えることを目指す。

```text
Webアプリ開発・データ分析・AI活用に関する開発機会や長期インターンに前向きで、制作物や活動内容を確認した上で連絡しやすい学生エンジニア。
```

Contactページは、サイト全体の最後の導線として、シンプルかつ分かりやすく設計する。
