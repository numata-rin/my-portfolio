## 1. このドキュメントの目的

このドキュメントは、現在のポートフォリオサイトをより実用的なものに改善するための方針をまとめるファイルである。

主に以下の目的で使用する。

* 現状のポートフォリオサイトの不足点を整理する
* 優先的に改善すべきページを明確にする
* Claude Codeに改善方針を正確に伝える
* 長期インターン応募や開発機会につながるポートフォリオへ改善する
* 今後追加したいページや機能を整理する

各ページの詳細な要件は、以下のファイルに記載する。

* `contents-overview.md`
* `Home-details.md`
* `About-details.md`
* `Works-details.md`
* `Skils-details.md`
* `Contact-details.md`

このファイルでは、サイト全体の改善点・優先順位・将来的な拡張方針を扱う。

---

## 2. 現状のポートフォリオサイトの課題

現在のポートフォリオサイトは、基本的なページ構成はできている。

現状の構成は以下を想定する。

```text
Home
About
Skills
Works
Experience
Contact
```

ただし、現状では以下の課題がある。

* 各ページの情報量がやや少ない
* 技術名は載っているが、何を実装できるのかが伝わりにくい
* Worksページで、制作物の背景・担当範囲・工夫点が不足している
* Aboutページで、研究関心や開発思想が十分に伝わっていない
* Contactページの連絡導線が弱い
* サイト全体として、長期インターン応募向けの説得力がまだ弱い
* 研究・開発・学内カンパニー活動のつながりがやや見えにくい

そのため、単なる自己紹介サイトから、以下のような実用的なポートフォリオサイトへ改善する。

```text
自己紹介サイト
↓
制作物を見せるポートフォリオ
↓
研究・開発・学内活動をつなげたポートフォリオ
↓
長期インターンや開発機会に応募できる実用的なポートフォリオ
```

---

## 3. 改善の基本方針

改善の基本方針は以下である。

### 3.1 技術名だけで終わらせない

現状のSkillsページでは、技術名の一覧だけでは閲覧者に実力が伝わりにくい。

悪い例：

```text
React / Next.js / Django / Python
```

良い例：

```text
Reactでは、コンポーネント分割、状態管理、フォーム処理、LocalStorageを用いたデータ保存などを実装経験があります。
```

技術名だけでなく、以下を明記する。

* その技術で何を作ったのか
* どの範囲を実装したのか
* どのような機能を扱ったのか
* 今後どのように伸ばしていくのか

---

### 3.2 制作物の背景・工夫点を明確にする

Worksページでは、作品名・技術スタックだけでは不十分である。

各作品について以下を記載する。

* 何を作ったのか
* なぜ作ったのか
* どの技術を使ったのか
* 自分は何を担当したのか
* どのような工夫をしたのか
* どこで苦労したのか
* 今後どのように改善したいのか
* GitHubやDemo URLはあるか

これにより、閲覧者が「この人は実際に何ができるのか」を判断しやすくなる。

---

### 3.3 研究・開発・学内活動をつなげる

自分の強みは、単にWebアプリを作っているだけでなく、以下の要素がつながっている点である。

* 情報工学を学んでいる
* React / Next.js / Django / Python でWebアプリを開発している
* 情報推薦・行動支援・学習支援に関心がある
* 学内カンパニーで実務に近い開発経験を積んでいる
* 将来的にAI・データ分析を活用したプロダクト開発を目指している

この流れがサイト全体で伝わるようにする。

```text
Webアプリ開発
↓
学習支援・行動支援への関心
↓
情報推薦・データ分析・AI活用
↓
人の意思決定や学習を支援するプロダクト開発
```

---

### 3.4 誇張せず、成長過程を伝える

未経験の技術や未完成の作品を過度に大きく見せない。

ただし、学習中・改善中・今後追加予定であることを明確に書くことで、成長意欲を伝える。

悪い例：

```text
AI開発の専門家です。
```

良い例：

```text
Pythonやデータ分析の基礎を学びながら、情報推薦・行動支援分野への応用に関心を持っています。
```

ポートフォリオ全体として、現在の実力と今後の方向性が正直に伝わる文章にする。

## 5. 各ページごとの改善方針

## 5.1 Homeページ

Homeページは、サイトの入口として、閲覧者に「何者か」を一瞬で伝える必要がある。

改善方針は以下である。

* 肩書きを明確にする
* 名前を分かりやすく表示する
* 短い自己紹介を追加する
* 現在取り組んでいることを簡潔に伝える
* WorksやGitHubへの導線を置く
* 詳細説明を詰め込みすぎない

Homeでは、以下のメッセージが伝わるようにする。

```text
情報工学を学びながら、React / Next.js / Django / Python を用いたWebアプリ開発に取り組んでいる学生エンジニアです。
```

---

## 5.2 Aboutページ

Aboutページは、人柄・関心・開発姿勢を伝えるページである。

* 開発で大切にしている考え方を書く

将来的に追加を検討したい機能は以下である。

---

## 7.1 作品詳細ページへの遷移

Worksページの各作品カードから、詳細ページへ遷移できるようにする。

例：

```text
Works一覧
```

各作品に対して、以下のリンクを分かりやすく設置する。

* GitHub
* Demo
* README
* 関連記事

リンクがない作品については、無理にボタンを表示せず、必要に応じて「準備中」とする。

---

## 7.3 画像・スクリーンショットの追加

WorksページやWorks Detailページには、作品のスクリーンショットを追加する。

スクリーンショットがあることで、閲覧者は作品内容を直感的に理解しやすくなる。

画像は以下を意識する。

* 容量を抑える
* 画質を保つ
* 表示サイズに合わせてリサイズする
* 必要に応じてWebPやNext.jsの画像最適化を使う

---

## 7.4 レスポンシブ対応の確認

スマートフォンやタブレットでも見やすいように、以下を確認する。

* ナビゲーションが崩れていないか
* Worksカードが見やすいか
* 文字サイズが小さすぎないか
* 画像が大きすぎないか
* 横スクロールが発生していないか

---

## 7.5 データ管理の整理

WorksやSkillsの情報は、今後増える可能性がある。

そのため、コード内に直接ベタ書きするより、配列やオブジェクトで管理できるようにするとよい。

例：

```ts
const works = [
  {
    title: "Study Tasks Manager",
    description: "学習タスクを管理するReactアプリ",
    techStack: ["React", "TypeScript", "Vite"],
    githubUrl: "...",
    demoUrl: "...",
  },
];
```

## 9. Claude Codeへの実装指示

Claude Codeには、このファイルを改善方針として読み込ませる。

実装時には、以下の方針に従う。

* 現状のデザインを大きく崩さない
* まずは情報量と伝わりやすさを改善する
* WorksとSkillsを最優先で改善する
* 各ページの詳細要件は個別ファイルを参照する
* 文章は過度に誇張しない
* 実装済みの内容と今後の予定を分けて書く
* 未取得の資格や未経験の技術を断定的に書かない
* 追加しやすいデータ構造を意識する
* Next.js / React / TypeScript の構成を崩さない
* Vercelでのデプロイを前提にする

## 12. まとめ

現状のポートフォリオサイトは、ページ構成の土台はできている。

しかし、長期インターンや開発機会につなげるためには、以下を重点的に改善する必要がある。

* Worksページで制作物の背景・機能・担当範囲・工夫点を詳しく伝える
* Skillsページで技術名だけでなく、実装経験を伝える
* Aboutページで研究関心・開発思想・学内カンパニーでの経験を伝える
* Contactページで連絡導線を強化する
* 将来的にResearchページ・Blog / Notesページ・Works Detailページを追加する

まずは、既存ページの改善を優先する。

その後、作品詳細ページや研究ページを追加し、研究・開発・学内活動がつながった実用的なポートフォリオサイトへ発展させる。
