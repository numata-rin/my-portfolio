* `Home-details.md`

## 2. ポートフォリオサイトの目的

## 10. サイト全体の文章方針

文章は、以下の方針で作成する。

### 10.1 誇張しすぎない

実務経験や技術力を過度に大きく見せるのではなく、現在の学習段階・実装経験・今後の改善予定を正直に書く。

悪い例：

```text
AI開発の専門家です。
```

良い例：

```text
Pythonやデータ分析の基礎を学びながら、情報推薦・行動支援分野への応用に関心を持っています。
```

### 10.2 技術名だけで終わらせない

悪い例：

```text
React / Django / Python が使えます。
```

良い例：

```text
Reactでは、コンポーネント分割、状態管理、フォーム処理、LocalStorageを用いたデータ保存などを実装しました。
```

### 10.3 制作背景を書く

作品ごとに、なぜ作ったのかを記載する。

悪い例：

```text
Todoアプリを作りました。
```

良い例：

```text
学習計画を自分で管理する中で、タスクを可視化し、学習の継続を支援する仕組みを作ることを目的に開発しました。
```

### 10.4 今後の改善点を書く

完成度が高くない作品でも、改善予定を明記することで、成長意欲や設計意識を伝える。

例：

```text
今後はDjango REST Frameworkと連携し、ログイン機能やDB保存、学習履歴の分析機能を追加する予定です。
```

## 13. Claude Codeへの実装指示方針

Claude Codeに実装を依頼する際は、この `contents-overview.md` をサイト全体の要件定義として参照する。

各ページの具体的な修正・追加内容は、以下の詳細ファイルを参照する。

```text
Home-details.md
About-details.md
Works-details.md
Skils-details.md
Contact-details.md
Improvement-point.md
```

Claude Codeには、以下の方針で実装を依頼する。

* このファイルをサイト全体の方針として扱う
* 各ページ詳細ファイルに従ってページ内容を修正する
* 既存のデザインを大きく崩さず、情報量を増やす
* 文章を読みやすいセクションに分ける
* WorksとSkillsを特に重点的に改善する
* 今後作品を追加しやすいデータ構造にする
* 必要に応じてコンポーネント分割を行う
* Vercelでのデプロイを前提に、Next.jsの構成を崩さない
