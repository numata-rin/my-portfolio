# Aboutページ 要件定義

## 1. このドキュメントの目的

このドキュメントは、ポートフォリオサイトの `About` ページに掲載する内容・役割・改善方針・実装方針を定義するための要件定義ファイルである。

Aboutページは、単なるプロフィール紹介ではなく、以下を伝えるためのページである。

* 自分がどのような学生なのか
* どのような分野に関心があるのか
* なぜWebアプリ開発に取り組んでいるのか
* 開発で何を大切にしているのか
* 学内カンパニーでどのような活動をしているのか
* 将来的にどのようなエンジニアを目指しているのか

Homeページでは短く概要を伝え、Aboutページではその背景や考え方を少し詳しく説明する。

---

## 2. Aboutページの役割

Aboutページの役割は、閲覧者に対して「この人がどのような考えで学習・開発・研究に取り組んでいるのか」を伝えることである。

Homeページが「一瞬で何者かを伝えるページ」であるのに対して、Aboutページは「なぜその分野に取り組んでいるのか」「どのような価値観を持っているのか」を伝えるページである。

特に、以下の印象を与えることを目指す。

```text
情報工学を学びながら、Webアプリ開発・データ分析・情報推薦を通じて、人の行動や学習を支援するプロダクトづくりに取り組んでいる学生エンジニア
```

---

## 3. Aboutページで伝えたいこと

Aboutページでは、以下を伝える。

* 岩手大学で情報工学を学んでいること
* Webアプリ開発に取り組んでいること
* 情報推薦・行動支援・学習支援に関心があること
* 課題を構造化し、使いやすい形に落とし込むことを大切にしていること
* 学内カンパニーでWebアプリ開発を担当していること
* 研究と開発をつなげたプロダクト開発に関心があること
* 将来的に、AI・データ分析・Web開発を組み合わせたプロダクトを作りたいこと

---

Aboutページは、以下の構成を基本とする。

```text

```

## 12. Aboutページに載せすぎない方がよい内容

Aboutページには、以下を詳しく書きすぎない。

* 制作物の詳細説明
* 技術スタックの細かい説明
* 資格一覧
* GitHubリポジトリ一覧
* 長すぎる研究説明
* 具体的な実装コード
* 過度な自己PR

これらは以下のページに分ける。

```text
制作物の詳細 → Works
技術スタック → Skills
資格・活動歴 → Experience
研究の詳細 → 将来的なResearchページ
連絡先 → Contact
```

Aboutページでは、あくまで「自分の背景・関心・価値観」を伝える。

## 15. 実装方針

Aboutページの実装では、以下を意識する。

### 15.1 文章をセクションに分ける

長い文章を1つの段落にまとめるのではなく、以下のようにセクションを分ける。

```text
自己紹介
関心のある分野
開発で大切にしていること
現在の活動
今後目指したいこと
```

セクションに分けることで、閲覧者が内容を理解しやすくなる。

---

### 15.2 カード形式や見出しを活用する

情報量が多くなる場合は、カード形式で整理する。

例：

```text
Web Development
React / Next.js / Django を用いたWebアプリ開発

Research Interest
情報推薦・行動支援・学習支援に関心

Campus Company
学内カンパニーでWebアプリ開発を担当

Future Vision
AI・データ分析を活用したプロダクト開発を目指す
```

---

### 15.3 Homeページとの差別化

Homeページと同じ内容を繰り返しすぎない。

Homeでは短く概要を伝え、Aboutでは背景や考え方を詳しく伝える。

```text
Home:
何者かを短く伝える

About:
なぜその分野に取り組んでいるのか、何を大切にしているのかを伝える
```

---

### 15.4 Skillsページとの差別化

Aboutページでは、技術名を並べすぎない。

技術の詳細説明はSkillsページに任せる。

Aboutページでは、技術そのものよりも、以下を重視する。

* なぜ開発しているのか
* どのような課題に関心があるのか
* 技術をどう活用したいのか
* どのようなエンジニアを目指しているのか

---

### 15.5 Worksページとの差別化

Aboutページでは、制作物の詳細を書きすぎない。

制作物の詳細はWorksページに任せる。

Aboutページでは、制作物に取り組む背景や姿勢を伝える。

---

## 16. Aboutページで使用するリンク

Aboutページから、必要に応じて以下のページへリンクを設置する。

```text
/works
/skills
/contact
```

リンクの例：

```text
制作物を見る → /works
使用技術を見る → /skills
連絡する → /contact
```

ただし、リンクを置きすぎるとページが散らかるため、必要最小限にする。

## 17. Aboutページの完成条件

Aboutページの完成条件は以下とする。

* 自分が情報工学を学んでいることが伝わる
* Webアプリ開発に取り組んでいることが伝わる
* 情報推薦・行動支援・学習支援に関心があることが伝わる
* 開発で大切にしている考え方が伝わる
* 学内カンパニーでの活動が伝わる
* 将来的に目指している方向性が伝わる
* Homeページよりも少し詳しい自己紹介になっている
* SkillsやWorksの内容と重複しすぎていない
* 文章が長すぎず、読みやすいセクションに分かれている
* 技術力や経験を過度に誇張していない
* スマートフォンでも読みやすい

## 18. Claude Codeへの実装指示

Claude Codeには、以下の方針でAboutページを修正してもらう。

```text
About-details.mdを参照し、Aboutページを自己紹介・研究関心・開発思想・現在の活動・将来像が伝わるページに改善してください。

Homeページでは概要を短く伝えているため、Aboutページでは「なぜWebアプリ開発に取り組んでいるのか」「どのような分野に関心があるのか」「開発で何を大切にしているのか」が伝わるようにしてください。

掲載内容は、自己紹介、関心のある分野、開発で大切にしていること、現在の活動、今後目指したいことのセクションに分けて整理してください。

既存デザインを大きく崩さず、文章量が多くなりすぎないように、見出し・カード・余白を使って読みやすくしてください。

技術力や経験を過度に誇張せず、現在の学習段階と今後の方向性が正直に伝わる表現にしてください。
```

---

## 19. 注意事項

* Aboutページを長くしすぎない
* Homeページと同じ文章をそのまま繰り返さない
* Skillsページのように技術一覧だけにしない
* Worksページのように制作物を詳しく説明しすぎない
* 未経験・学習中の技術を断定的に書かない

## 20. まとめ

Aboutページは、ポートフォリオサイトの中で「人柄・関心・開発姿勢」を伝えるページである。

このページでは、以下を明確に伝える。

```text
岩手大学で情報工学を学びながら、Webアプリ開発・データ分析・情報推薦に関心を持ち、人の行動や学習を支援するプロダクトづくりを目指していること。
```

また、開発では以下を大切にしていることを伝える。

```text
単に機能を実装するだけでなく、誰のどのような課題を解決するのかを考え、使いやすい形に落とし込むこと。
```

Aboutページを通して、閲覧者に以下の印象を与えることを目指す。

```text
技術を学ぶだけでなく、課題解決や人の行動支援につなげようとしている学生エンジニア
```
