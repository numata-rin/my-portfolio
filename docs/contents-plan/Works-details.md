# Works-details.md

# Worksページ 要件定義

## 1. このドキュメントの目的

このドキュメントは、ポートフォリオサイトの `Works` ページに掲載する内容・役割・改善方針・実装方針を定義するための要件定義ファイルである。

Worksページは、ポートフォリオサイトの中で最も重要なページである。

理由は、閲覧者が以下を判断するために、制作物を最も重視するためである。

* 実際に何を作ったのか
* どの技術を使ったのか
* どこまで自分で実装したのか
* どのような意図で開発したのか
* どのような工夫をしたのか
* 今後どのように改善していく予定なのか

そのため、Worksページでは、単に作品名や技術スタックを並べるだけでなく、各制作物について背景・機能・担当範囲・工夫点・改善予定まで整理して記載する。

---

## 2. Worksページの役割

Worksページの役割は、閲覧者に対して「自分が実際に作ったもの」と「開発に対する考え方」を伝えることである。

特に、長期インターンや開発機会に応募する場合、Worksページでは以下を見られる。

* Webアプリを実際に完成させた経験があるか
* React / Next.js / Django / Python などをどのように使っているか
* GitHubやVercelなどを使って公開・管理しているか
* READMEやIssueを使って開発内容を整理しているか
* ただ作っただけでなく、目的や改善点を説明できるか
* 個人開発・授業開発・学内カンパニーでの開発経験があるか

Worksページは、スキルを言葉で説明するだけでなく、制作物を通して技術力・設計意識・学習姿勢を示すページである。

---

## 3. Worksページで伝えたいこと

Worksページでは、以下を伝える。

* React / TypeScript / Vite を用いた個人開発経験があること
* Djangoを用いたバックエンド開発経験があること
* Next.js / Vercel を用いたポートフォリオサイト制作経験があること
* 学内カンパニーで実務に近いプロトタイプ開発に取り組んでいること
* 制作物ごとに目的・背景・機能・工夫点を説明できること
* 今後、API連携・DB保存・認証・データ分析機能へ発展させる意欲があること
* 研究関心である情報推薦・行動支援・学習支援と、今後の制作物をつなげていく予定であること


# 21. 実装方針

## 21.1 Worksカードをコンポーネント化する

```ts
type WorkCardProps = {

  futureImprovements?: string;
```

## 21.2 Worksデータを配列で管理する

制作物情報は、配列で管理する。

例：

```ts
const works = [
  {

      "学習タスクを追加・表示・完了切り替え・削除できるReact製のタスク管理アプリです。",
```

# 26. まとめ

Worksページでは、以下を伝える。

```text
React / Next.js / Django / Python などを用いて、実際にWebアプリやプロトタイプを制作していること。
```

また、単に制作物を並べるのではなく、以下を明確にする。

```text
何を作ったのか
なぜ作ったのか
どの技術を使ったのか
どこを担当したのか
どのような工夫をしたのか
今後どのように改善したいのか
```

Worksページを通して、閲覧者に以下の印象を与えることを目指す。

```text
Webアプリ開発の基礎を実際の制作物として形にし、目的・設計・改善点まで説明できる学生エンジニア。
```

そのため、Worksページはこのポートフォリオサイトの中で最も重点的に改善するページとして扱う。
