# Homeページ 要件定義

## 1. このドキュメントの目的

このドキュメントは、ポートフォリオサイトの `Home` ページに掲載する内容・役割・改善方針・実装方針を定義するための要件定義ファイルである。

Homeページは、ポートフォリオサイトの入口となるページであり、閲覧者が最初に見る可能性が最も高いページである。

そのため、Homeページでは細かい説明を長く書くのではなく、以下を短時間で伝えることを目的とする。

* 自分が何者なのか
* 何を学んでいるのか
* 何に取り組んでいるのか
* どのような分野に関心があるのか
* どのページを見てほしいのか

詳細な自己紹介は `About` ページ、制作物の説明は `Works` ページ、技術スタックは `Skills` ページに任せる。

Homeページでは、サイト全体への導線として機能することを重視する。

---

## 2. Homeページの役割

Homeページの役割は、閲覧者に対して「この人はどのような学生エンジニアなのか」を一瞬で伝えることである。

特に、以下の印象を与えることを目指す。

```text
情報工学を学びながら、Webアプリ開発・データ分析・情報推薦に関心を持ち、人の行動や学習を支援するプロダクトづくりに取り組んでいる学生エンジニア
```

Homeページでは、以下の流れが自然に伝わるようにする。

```text
名前
↓
肩書き
↓
短い自己紹介
↓
現在取り組んでいること
↓
Works / About / GitHub への導線
```

---

## 3. Homeページで伝えたいこと

Homeページで伝えたい内容は以下である。

* 岩手大学で情報工学を学んでいること
* React / Next.js / Django / Python を用いたWebアプリ開発に取り組んでいること
* 情報推薦・行動支援・学習支援に関心があること
* 学内カンパニーでWebアプリ開発に取り組んでいること
* 将来的に、Web開発・データ分析・AIを活用したプロダクト開発を目指していること
* 制作物やGitHubを見てほしいこと

ただし、Homeページでは情報を詰め込みすぎない。

詳細説明は各ページに分け、Homeではあくまで概要と導線を中心にする。

---

## 4. Homeページの基本構成

Homeページは、以下の構成を基本とする。

```text
Hero Section
Profile Summary Section
Current Focus Section
Navigation / CTA Section
```

または、よりシンプルに以下の構成でもよい。

```text
Hero Section
CTA Buttons
Short Overview
```

既存デザインとの整合性を優先し、無理に大きなレイアウト変更は行わない。

---

## 5. Hero Section

Hero Sectionは、Homeページで最も重要な部分である。

閲覧者がページを開いて最初に見るエリアであり、ここで自分が何者なのかを明確に伝える。

### 5.1 Hero Sectionに掲載する内容

Hero Sectionには以下を掲載する。

* 肩書き
* 名前
* 短いキャッチコピー
* 2〜3文程度の自己紹介
* Works / GitHub / About へのボタン

---

### 5.2 肩書き案

肩書きは、以下のような表現を検討する。

```text
学生エンジニア / Web Developer
```

または、

```text
Information Engineering Student / Web Developer
```

または、

```text
Webアプリ開発と情報推薦に取り組む学生エンジニア
```

長期インターン向けには、以下の表現が最も分かりやすい。

```text
学生エンジニア / Web Developer
```

---

### 5.3 名前の表示

名前は、以下のように表示する。

```text
沼田 倫
```

必要に応じてローマ字も併記する。

```text
沼田 倫 / Rin Numata
```

グローバルな印象やGitHubとの整合性を考える場合は、ローマ字併記が望ましい。

---

### 5.4 キャッチコピー案

Homeページのメインコピーとして、以下のいずれかを使用する。

#### 案1：標準案

```text
情報工学を学びながら、Webアプリ開発・データ分析・情報推薦に取り組んでいます。
```

#### 案2：研究関心を強める案

```text
Webアプリ開発を軸に、情報推薦・行動支援・学習支援へ関心を広げています。
```

#### 案3：プロダクト志向を強める案

```text
人の行動や学習を支援するプロダクトづくりを目指して、Web開発とデータ分析を学んでいます。
```

#### 案4：長期インターン向け案

```text
React / Next.js / Django / Python を用いて、Webアプリ開発に取り組む情報工学系の学生エンジニアです。
```

最もおすすめするメインコピーは以下である。

```text
Webアプリ開発を軸に、情報推薦・行動支援・学習支援へ関心を広げています。
```

理由は、Web開発と研究関心の両方が短く伝わるためである。

## 7. CTAボタン

Homeページには、閲覧者が次に見るべきページへ移動できるボタンを配置する。

CTAは、Homeページの重要な役割である。

### 7.1 設置するボタン

最低限、以下のボタンを設置する。

```text
Worksを見る
GitHubを見る
```

可能であれば、以下も追加する。

```text
Aboutを見る
Contact
```

ただし、ボタンが多すぎると分かりにくくなるため、メインCTAは2つに絞る。

---

### 7.2 CTAボタンの優先順位

優先順位は以下とする。

```text
1. Worksを見る
2. GitHubを見る
3. Aboutを見る
4. Contact
```

最も重要なのは `Worksを見る` である。

長期インターンや開発機会では、制作物を見てもらうことが重要であるため、HomeからWorksページへの導線を目立たせる。

---

### 7.3 CTAボタンの文言案

```text
Worksを見る
GitHubを見る
```

または、

```text
制作物を見る
GitHubを見る
```

より自然な日本語にするなら、以下がおすすめである。

```text
制作物を見る
GitHubを見る
```

英語寄りのデザインにする場合は、以下でもよい。

```text
View Works
View GitHub
```

サイト全体が日本語中心の場合は、日本語表記を推奨する。

Hero Sectionの下に、簡単なプロフィール概要を置いてもよい。

ただし、Aboutページと重複しすぎないように注意する。

以下のような短いカードを配置する。

```text
岩手大学 理工学部 情報工学系
```

---

### 8.2 カード形式の表示案

```text

情報工学・Web開発・データ分析を学習中

React / Next.js / Django を用いたWebアプリを開発

Research Interest
```

このように3つ程度のカードに分けると、視覚的に分かりやすい。

## 9. Current Focus Section

現在取り組んでいることを簡潔に示すセクションを追加してもよい。

### 9.1 掲載する内容

```text
現在取り組んでいること

- React / Next.js を用いたポートフォリオサイト改善
- Django REST Frameworkを用いたAPI開発の学習
- 情報推薦・行動支援分野の研究準備
- 学内カンパニーでのWebアプリ開発
```

---

### 9.2 表示上の注意

Current Focus Sectionは、Homeページに必須ではない。

Homeが長くなりすぎる場合は、AboutやExperienceに移動してもよい。

ただし、閲覧者に「現在進行形で動いている人」という印象を与えたい場合は有効である。

## 10. Homeページに載せすぎない方がよい内容

Homeページでは、以下の内容を詳しく書きすぎない。

* 技術スタックの詳細説明
* 各制作物の詳しい説明
* 研究内容の長い説明
* 資格一覧
* 学歴の詳細
* 長い自己PR文
* 開発思想の詳細

これらは以下のページに分ける。

```text
技術スタックの詳細 → Skills
制作物の詳細 → Works
自己紹介・開発思想 → About
活動歴・資格 → Experience
連絡先 → Contact
```

Homeページは、あくまでサイト全体への入口として設計する。

## 13. 実装方針

Homeページの実装では、以下を意識する。

### 13.1 既存デザインを大きく崩さない

現状のポートフォリオサイトのデザインを尊重する。

大幅なレイアウト変更よりも、まずは文章・導線・情報整理を改善する。

---

### 13.2 Hero Sectionを分かりやすくする

Hero Sectionには、以下を明確に表示する。

* 肩書き
* 名前
* キャッチコピー
* 短い自己紹介
* CTAボタン

視線の流れが自然になるように、情報の優先順位を意識する。

---

### 13.3 CTAボタンを目立たせる

`制作物を見る` ボタンを最も目立たせる。

`GitHubを見る` ボタンは、2番目の導線として配置する。

ボタンの遷移先は以下とする。

```text
制作物を見る → /works
GitHubを見る → GitHubプロフィールURL
```

GitHubプロフィールURLは、実際のアカウントURLに置き換える。

---

### 13.4 レスポンシブ対応

スマートフォンでも見やすくする。

確認する点は以下である。

* Hero Sectionの文字が大きすぎないか
* CTAボタンが縦並びでも見やすいか
* 余白が狭すぎないか
* 横スクロールが発生していないか
* 長い文章が読みづらくなっていないか

---

### 13.5 テキストの管理

今後文章を変更しやすいように、可能であれば表示テキストを定数として整理する。

例：

```ts
const heroContent = {
  label: "学生エンジニア / Web Developer",
  name: "沼田 倫 / Rin Numata",
  catchCopy: "Webアプリ開発を軸に、情報推薦・行動支援・学習支援へ関心を広げています。",
  description: [
    "岩手大学で情報工学を学びながら、React / Next.js / Django / Python を用いたWebアプリ開発に取り組んでいます。",
    "現在は、人の学習や行動を支援するプロダクト開発を目指し、個人開発や学内カンパニーでの開発活動を進めています。",
  ],
};
```

## 15. Homeページの完成条件

Homeページの完成条件は以下とする。

* ページを開いてすぐに何者かが分かる
* 名前・肩書き・短い自己紹介が表示されている
* Webアプリ開発と研究関心の方向性が伝わる
* Worksページへの導線がある
* GitHubへの導線がある
* 文章が長すぎず、読みやすい
* スマートフォンでも見やすい
* About / Works / Skills など他ページと内容が重複しすぎていない
* 現状のデザインを大きく崩していない

---

## 16. Claude Codeへの実装指示

Claude Codeには、以下の方針でHomeページを修正してもらう。

```text
Home-details.mdを参照し、Homeページをポートフォリオサイトの入口として改善してください。

目的は、閲覧者が最初に見たときに、沼田倫がどのような学生エンジニアなのかを短時間で理解できるようにすることです。

既存デザインを大きく崩さず、Hero Sectionを中心に、肩書き・名前・キャッチコピー・短い自己紹介・CTAボタンを整理してください。

CTAボタンは「制作物を見る」と「GitHubを見る」を優先してください。

Homeページでは詳細を書きすぎず、詳しい自己紹介はAbout、制作物はWorks、技術スタックはSkillsへ誘導する構成にしてください。
```

---

## 17. 注意事項

* Homeページに情報を詰め込みすぎない
* 自己紹介文を長くしすぎない
* 技術力を過度に誇張しない
* 未経験・学習中の内容を断定的に書かない
* WorksページやSkillsページと内容を重複させすぎない
* まずは閲覧者に「何者か」が伝わることを優先する
* CTAボタンを分かりやすく配置する
* 既存のデザインや雰囲気を大きく崩さない
