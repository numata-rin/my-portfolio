# Skillsページ 変更・追加提案

## 提案① 「Currently Learning」セクションの追加

### 内容
学習中・今後取り組む予定の技術を、既存の技術タグ一覧とは別のセクションとして末尾に追加する。

### 対象技術（Currently Learning に移動または新規追加）
- Django REST Framework（現在バックエンドに含まれているが実装経験が浅い）
- TensorFlow（現在データサイエンスに含まれているが実装経験が浅い）
- scikit-learn
- 推薦システム
- LLM活用
- API設計 / PostgreSQL / 認証機能

### 理由
- 現状は「使える技術」と「学習中の技術」が同じセクションに混在している
- TensorFlow や Django REST Framework が「実装経験あり」と誤読される可能性がある
- 長期インターンの採用担当者に対して、現在の実力と今後の方向性を正直に伝えることが重要
- 「Currently Learning」を設けることで、自分の成長意欲・学習方向性も同時に伝えられる

## 提案② 各カテゴリへの説明文追加

### 内容
現在タグ一覧のみの各カテゴリに、1〜2文の経験内容・学習状況の説明文を追加する。

### 各カテゴリの説明文案（Skils-details.md 短縮版より）

**フロントエンド**
> Reactでは、コンポーネント分割、状態管理、フォーム処理、LocalStorage保存などを実装しました。
> Next.jsでは、ポートフォリオサイトやプロトタイプ開発を通して、ページ構成やVercelへのデプロイを経験しています。

**バックエンド**
> Djangoでは、モデル設計、URL設計、ビュー処理、フォーム処理、テンプレート連携などを経験しました。
> 今後はDjango REST Frameworkを用いたAPI設計や、React / Next.jsとの連携を重点的に学習していく予定です。

**データサイエンス**
> Pythonを用いたデータ分析・機械学習の基礎を学習しています。
> NumPy / Pandasによるデータ処理、Matplotlibによる可視化の基礎に取り組んでいます。
> 将来的には、情報推薦・行動支援・学習支援への応用を目指しています。

**ツール / インフラ**
> GitHubを用いたバージョン管理、Vercelを用いたデプロイ、Notionを用いたタスク管理・要件整理を経験しています。

### 理由<li></li><li></li>
- 現状はタグ一覧のみで「Reactで何を作ったか」「Djangoで何を経験したか」が全く伝わらない
- 採用担当者が知りたいのは技術名ではなく「その技術で何ができるか」
- 説明文を加えることで、技術力の中身と学習の深さが具体的に伝わる
- 1〜2文に絞ることでページが長くなりすぎず、読みやすさも維持できる

---

## 変更しなくてよいと判断した点

- **アイコン付きタグのデザイン** — 視覚的に分かりやすく、現状でも十分機能している
- **ページ全体のレイアウト・構造** — シンプルで読みやすく、大きな変更は不要
- **関連制作物へのリンク** — 現状の制作物数では追加しても効果が薄い。Worksが充実した段階で検討
